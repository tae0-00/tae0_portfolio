FIG.01 · About

소개

기능을 만드는 것을 넘어, 기획과 데이터 구조를 함께 고민할 때 완성도 높은 서비스가 만들어진다 믿습니다.

기획만 하는 사람을 넘어 직접 만드는 사람이 되고 싶어 풀스택 개발에 뛰어들었습니다. 두 번의 팀 프로젝트에서 사용자 관점에서 기능을 생각하고, 문제 발생 시 DB 구조와 로직을 직접 재설계하는 과정에 가장 몰입했습니다.

같은 기능도 데이터를 어떤 구조로 두느냐에 따라 완전히 달라진다는 것 — 눈에 보이는 증상이 아니라 구조에서 원인을 찾아 해결하는 개발자입니다.

프로젝트

각 프로젝트의 개요와 개발 환경입니다. 프로젝트별 문제 해결 기록과 데이터 구조(ERD)는 상세 페이지에서 확인할 수 있습니다.

MAIN 전북로 (路) 2026.04.21 — 06.02 · 팀 프로젝트
담당 : 메인 · 여행지 목록/상세 · 역사 코스 · 관리자 전체

단순히 관광지를 나열하는 기존 플랫폼과 달리, 전북의 문화유산을 '역사'라는 콘텐츠로 엮어 탐방 경험을 제공하는 웹 플랫폼입니다.

전북 행정구역 GeoJSON을 SVG 지도로 직접 가공해 지역 기반 탐색 흐름을 설계했고, 공공 관광 API를 불러와 여행지 화면을 구성했습니다. 자체 제작 역사 코스와 공공 데이터를 분리·연동하는 구조를 적용하고, 회원·코스·신고·이미지 관리를 관리자 페이지 하나로 통합했습니다.

SUB 집사상점 2026.03.03 — 04.07 · 팀 프로젝트
담당 : 추천 · 상품 페이지 · 커뮤니티 · 관리자(주문/취소·신고)

반려동물의 종류와 연령을 기반으로 맞춤 상품을 추천하고, 반려인 간 소통이 가능한 커뮤니티를 결합한 쇼핑몰 플랫폼입니다.
